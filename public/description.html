Endpoints documentation

1. Get a user details by id

GET /api/users/:id

  • Parameters: :id (required) - User's unique identifier.
  • Description: fetches detailed information about a single user by their unique ID.

Example Response:

                        {
                            "user":  {
                                "user_id": 1,
                                "first_name": "Bill",
                                "surname": "Farrel",
                                "email": "bill.farrel@gmail.com",
                                "phone_number": "07456116735",
                                "avatar": "https://example.com/avatar.jpg">
                                "created_at": "2024-01-01T09:30:00Z"
                                }
                        }

Users ids type:

NAME DESCRIPTION TYPE
user_id The unique identifier for the user. integer
first_name The first name of the user. string
surname The surname (last name) of the user. string
email The email address of the user. string
phone_number The contact phone number of the user. string
avatar URL to the user's avatar image. string
created_at Timestamp indicating when the user was created (ISO 8601 format). string

2. Update a user's details by id

PATCH /api/users/:id

  • Parameters: :id (required) - User's unique identifier.
  • Request Body: JSON payload containing any combination of the following properties to update: "first_name", "surname", "email", "phone", "avatar".
  • Description: Updates the user details in the database for the specified user ID with the fields provided in the payload.

Example Payload:

                        {
                            "first_name": "Bill",
                            "surname": "Mille",
                            "email": "bill.mille@example.com",
                            "phone": "07453886735",
                            "avatar": "https://example.com/new-avatar.jpg"
                        }

Update user type:

NAME DESCRIPTION TYPE
user_id The unique identifier for the user. integer
first_name The first name of the user. string
surname The surname (last name) of the user. string
email The email address of the user. string
phone_number The contact phone number of the user. string
avatar URL to the user's avatar image. string
created_at Timestamp indicating when the user was created (ISO 8601 format). string

3. Get a user's bookings by id

GET /api/users/:id/bookings

  • Parameters: :id (required) - User's unique identifier.
  • Description: Fetches all bookings for the specified user ID, returned in chronological order of check-in dates.

Example Response:

            {
                "bookings": [
                    {
                        "booking_id": 1,
                        "check_in_date": "2025-01-10",
                        "check_out_date": "2025-01-15",
                        "property_id": 100,
                        "property_name": "Sunny Beach Resort",
                        "host": "John Doe",
                        "image": "https://example.com/property1.jpg"
                    },
                    {
                        "booking_id": 2,
                        "check_in_date": "2025-02-01",
                        "check_out_date": "2025-02-05",
                        "property_id": 101,
                        "property_name": "Mountain View Cabin",
                        "host": "Jane Smith",
                        "image": "https://example.com/property2.jpg"
                    }
                ]
            }

Bookings response type:

NAME DESCRIPTION TYPE
booking_id The unique identifier for the booking. integer
check_in_date The start date of the booking (ISO 8601 format). string
check_out_date The end date of the booking (ISO 8601 format). string
property_id The unique identifier for the property. integer
property_name The name of the property. string
host The name of the host responsible for the property. string
image The URL to the first image of the property. string
