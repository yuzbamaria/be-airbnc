Endpoints documentation

Users endpoints

1. Get a user details by id

GET /api/users/:id

  • Parameters: :id (required) - User's unique identifier.
  • Description: fetches detailed information about a single user by their unique ID.

Example Response:

    {
        "user":  {
            "user_id": 1,
            "first_name": "Bill",
            "surname": "Farrel",
            "email": "bill.farrel@gmail.com",
            "phone_number": "07456116735",
            "avatar": "https://example.com/avatar.jpg">
            "created_at": "2024-01-01T09:30:00Z"
            }
    }

Users ids type:

NAME DESCRIPTION TYPE
user_id The unique identifier for the user. integer
first_name The first name of the user. string
surname The surname (last name) of the user. string
email The email address of the user. string
phone_number The contact phone number of the user. string
avatar URL to the user's avatar image. string
created_at Timestamp indicating when the user was created (ISO 8601 format). string

2. Update a user's details by id

PATCH /api/users/:id

  • Parameters: :id (required) - User's unique identifier.
  • Request Body: JSON payload containing any combination of the following properties to update: "first_name", "surname", "email", "phone", "avatar".
  • Description: Updates the user details in the database for the specified user ID with the fields provided in the payload.

Example Payload:

    {
        "first_name": "Bill",
        "surname": "Mille",
        "email": "bill.mille@example.com",
        "phone": "07453886735",
        "avatar": "https://example.com/new-avatar.jpg"
    }

Update user type:

NAME DESCRIPTION TYPE
user_id The unique identifier for the user. integer
first_name The first name of the user. string
surname The surname (last name) of the user. string
email The email address of the user. string
phone_number The contact phone number of the user. string
avatar URL to the user's avatar image. string
created_at Timestamp indicating when the user was created (ISO 8601 format). string

3. Get a user's bookings by id

GET /api/users/:id/bookings

  • Parameters: :id (required) - User's unique identifier.
  • Description: Fetches all bookings for the specified user ID, returned in chronological order of check-in dates.

Example Response:

                "check_out_date": "2025-01-15",

                "property_name": "Sunny Beach Resort",
                "host": "John Doe",
                "image": "https://example.com/property1.jpg"
            },
            {
                "booking_id": 2,
                "check_in_date": "2025-02-01",
                "check_out_date": "2025-02-05",
                "property_id": 101,
                "property_name": "Mountain View Cabin",
                "host": "Jane Smith",
                "image": "https://example.com/property2.jpg"
            }
        ]
    }

Bookings response type:

NAME DESCRIPTION TYPE
booking_id The unique identifier for the booking. integer
check_in_date The start date of the booking (ISO 8601 format). string
check_out_date The end date of the booking (ISO 8601 format). string
property_id The unique identifier for the property. integer
property_name The name of the property. string
host The name of the host responsible for the property. string
image The URL to the first image of the property. string

Properties endpoints

4. Get all properties

GET /api/properties

  • Optional Queries:
    • ?maxprice=<max cost per night> – filters properties with a maximum cost per night.
    • ?minprice=<min cost per night> – filters properties with a minimum cost per night.
    • ?sort=<cost_per_night | popularity> – sorts properties by cost per night or popularity.
    • ?order=<ascending | descending> – specifies the order for sorting (ascending or descending).
    • ?host=<id> – filters properties by the specified host's ID.
  • Description: Fetches a list of properties, ordered by most favorited to least by default. Supports query parameters for filtering and sorting.

Example Response:

    {
        "properties": [
            {
                "property_id": 10,
                "property_name": "Sunny Beach Resort",
                "location": "Miami Beach, FL",
                "price_per_night": 200,
                "host": "Bill Mille"
            },
            {
                "property_id": 8,
                "property_name": "Cozy Mountain Cabin",
                "location": "Aspen, CO",
                "price_per_night": 150,
                "host": "Kate Smith"
            }
        ]
    }

Properties response type:

NAME DESCRIPTION TYPE
property_id The unique identifier for the property. integer
property_name The name of the property. string
location The location of the property (e.g., city or region). string
price_per_night The cost per night for the property. integer
host The name of the host responsible for the property. string

5. Favourite a property

POST /api/properties/:id/favourite

  • Parameters: :id (required) – property's unique identifier.
  • Payload:
    •     {
        "guest_id": <id>
    }
  • Description: Adds a new entry to the favourites table for the specified property and guest. This action marks the property as favourited by the user.

Example Response:

    {
        "msg": "Property favourited successfully.",
        "favorite_id": 25
    }

Favouriting response type:

NAME DESCRIPTION TYPE
msg A confirmation message indicating the action was successful. string
favorite_id The unique identifier for the newly created favourite record. integer

6. Get a specific property

GET /api/properties/:id

  • Parameters: :id (required) – the unique identifier of the property.
  • Optional Query:
    • ?user_id=<id> – indicates whether the user with the specified ID has favourited this property.
  • Description: Retrieves detailed information about a specific property. Optionally includes a flag indicating whether a specific user has marked the property as a favourite.

Example Response:

    {
        "property": {
            "property_id": 12,
            "property_name": "Charming Lakeside Cabin",
            "location": "Lake Tahoe, CA",
            "price_per_night": 300,
            "description": "A peaceful retreat with stunning lake views.",
            "host": "John Doe",
            "host_avatar": "https://example.com/images/host_avatar.jpg",
            "favourite_count": 5,
            "favourited": true
        }
    }

Property response type:

NAME DESCRIPTION TYPE
property_id The unique identifier for the property. integer
property_name The name of the property. string
location The property location (e.g., city or region). string
price_per_night The nightly cost of the property. integer
description A detailed description of the property. string
host The host's name responsible for the property. string
host_avatar The URL of the host's avatar image. string
favourite_count The number of times this property has been favourited. integer
favourited Indicates whether the specified user has marked the property as a favourite. boolean

7. Get reviews for a property

GET /api/properties/:id/reviews

  • Parameters: :id (required) – the unique identifier of the property.
  • Description: Retrieves all reviews for a specific property, sorted from latest to oldest by default. It also provides the average rating across all reviews.

Example Response:

    {
        "reviews": [
            {
                "review_id": 101,
                "comment": "Amazing property with great amenities.",
                "rating": 5,
                "created_at": "2025-01-01",
                "guest": "Jane Smith",
                "guest_avatar": "https://example.com/images/guest_avatar.jpg"
            },
            {
                "review_id": 102,
                "comment": "Good experience, but the location was a bit far.",
                "rating": 4,
                "created_at": "2024-12-25",
                "guest": "John Doe",
                "guest_avatar": "https://example.com/images/john_avatar.jpg"
            }
        ],
        "average_rating": 4.5
    }

Reviews response type:

NAME DESCRIPTION TYPE
reviews A list of review objects for the property. array
review_id The unique identifier for a review. integer
comment The comment provided by the reviewer. string
rating The rating (out of 5) provided by the reviewer. integer
created_at The date the review was submitted. string
guest The name of the guest who submitted the review. string
guest_avatar The URL of the guest's avatar image. string
average_rating The average rating of all reviews for the property. float

8. POST a new review

POST /api/properties/:id/reviews

  • Parameters: :id (required) – the unique identifier of the property being reviewed.
  • Request Body: Expects the following JSON payload: 
        {
        "guest_id": <id>,
        "rating": <rating>,
        "comment": <comment>
    }
  • Description: Inserts a new review for the specified property into the reviews table. The payload must include the guest ID, rating, and comment.

Example Response:

    {
        "review_id": 34,
        "property_id": 12,
        "guest_id": 45,
        "rating": 5,
        "comment": "Amazing place, loved the experience!",
        "created_at": "2025-01-05T12:34:56Z"
    }

Review response type:

NAME DESCRIPTION TYPE
review_id The unique identifier for the review. integer
property_id The unique identifier for the property being reviewed. integer
guest_id The unique identifier for the guest who submitted the review. integer
rating The rating given to the property by the guest. integer
comment The review text or comment provided by the guest. string
created_at The timestamp when the review was created. string

9. Get bookings for a specific property

GET /api/properties/:id/bookings

  • Parameters: :id (required) – the unique identifier of the property.
  • Description: Retrieves all bookings for the specified property, ordered from the latest check_out_date to the earliest.

Example Response:

    {
        "bookings": [
            {
                "booking_id": 34,
                "check_in_date": "2025-01-01",
                "check_out_date": "2025-01-05",
                "created_at": "2024-12-15T10:00:00Z"
            },
            {
                "booking_id": 30,
                "check_in_date": "2024-12-20",
                "check_out_date": "2024-12-25",
                "created_at": "2024-11-30T08:30:00Z"
            }
        ],
        "property_id": 12
    }

Booking response type:

NAME DESCRIPTION TYPE
booking_id The unique identifier for the booking. integer
check_in_date The date when the booking starts. string (date)
check_out_date The date when the booking ends. string (date)
created_at The timestamp when the booking was created. string (date)
property_id The unique identifier for the property associated with the bookings. integer

Reviews endpoints

11. Delete a review

DELETE /api/reviews/:id

  • Parameters: :id (required) – the unique identifier of the review to be deleted.
  • Description: This endpoint allows users to delete a specific review identified by its unique review ID from the reviews table. Responds with the status code 204, indicating that the review was successfully deleted.

12. Delete a booking

DELETE /api/bookings/:id

  • Parameters: :id (required) – the unique identifier of the booking to be deleted.
  • Description: This endpoint allows users to delete a specific booking identified by its unique booking ID from the bookings table. Responds with the status code 204, indicating that the booking was successfully deleted.

13. Update a booking

PATCH /api/bookings/:id

  • Parameters: :id (required) – the unique identifier of the booking to be updated.
  • Request Body: 
        {
        "check_in_date": ,
        "check_out_date": 
    }
    The request may include any combination of the `check_in_date` and `check_out_date` properties to update the corresponding values in the booking.
  • Description: This endpoint allows users to update specific properties of a booking, such as the check-in and check-out dates, using the unique booking ID.

Example Response:

    {
        "booking_id": 123,
        "check_in_date": "2025-03-15",
        "check_out_date": "2025-03-20",
        "created_at": "2025-01-06"
    }

Booking response type:

NAME DESCRIPTION TYPE
booking_id The unique identifier for the booking. integer
check_in_date The date of check-in for the booking. date
check_out_date The date of check-out for the booking. date
created_at The date and time when the booking was created. date
